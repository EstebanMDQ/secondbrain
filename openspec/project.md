# Project Context

## Purpose
A self-hosted Telegram bot for capturing and organizing project ideas on the go.
Chat naturally about projects, and the AI organizes notes into per-project
Obsidian-friendly markdown files synced via git. Access your ideas from any machine.

Distributed as an installable Python package with a CLI for setup and service
management. Targets Linux primarily and macOS secondarily.

## Tech Stack
- Python 3.13 + uv
- click (CLI framework)
- python-telegram-bot (async)
- SQLAlchemy (SQLite, JSON columns, create_all)
- openai Python package (OpenAI-compatible API for any provider)
- TOML config file (~/.config/second-brain/config.toml)
- systemd (Linux) / launchd (macOS) for service management
- Docker Compose as optional deployment path

## Project Conventions

### Code Style
- PEP8 formatting
- Type hints throughout
- No `# generated by AI` or similar attribution

### Architecture Patterns
- Single async Python process, no task queues
- Service layer pattern: handlers -> services -> store/sync
- Two-tier AI: cheap model for categorization, bigger model for discussion
- OpenAI-compatible API as the provider interface
- XDG Base Directory compliance for config and data

### Testing Strategy
- Unit tests: store CRUD/dedup, markdown generation, response parser, config loading
- Integration tests: chat handler with mocked AI, export with mocked DB
- pytest as test runner

### Git Workflow
- Concise commit messages
- Single main branch

## Domain Context
- Single-user personal tool
- The user captures ideas via Telegram while away from the computer
- Notes must be available in an Obsidian vault (git-synced) when working
- The user works across multiple machines
- Installs as a system service, runs in background

## Important Constraints
- Single-user only (configured via TOML or env var)
- AI provider agnostic via OpenAI-compatible API
- No database migrations - create_all on startup, SQLite is a disposable index
- Obsidian markdown files are the source of truth, not SQLite
- Linux primary, macOS secondary, no Windows support

## External Dependencies
- Telegram Bot API (via python-telegram-bot)
- OpenAI-compatible AI endpoints (Ollama for categorization, configurable for discussion)
- Git remote for Obsidian vault sync
